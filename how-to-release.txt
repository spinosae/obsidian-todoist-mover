from https://marcus.se.net/obsidian-plugin-docs/publishing/release-your-plugin-with-github-actions#use-standard-version-to-automatically-tag-your-release

1. Make sure commits since the last release use readable Conventional Commit prefixes such as `feat:` or `fix:`.
2. Run `npm test`.
3. Run `npm run build`.
4. Run `npm run release`.
5. Push the release commit and tags with `git push --follow-tags origin master`.
6. Wait for the GitHub Action release workflow to finish.
